# SpeakAI — full activation plan

> **Goal**: turn the current UI-rich Electron app into a real multi-language
> social learning platform with cross-user content, real cloud AI, and real
> exam scoring. Today every page renders, ~95% of UI is in place, but the
> backend is local-only and the AI integration is a contract waiting to be
> wired.
>
> This document is the single source of truth for what's left to ship.
> Updated whenever the user requests a feature.

## 0 · Diagnostic snapshot — what we have (2026-05-28)

### Frontend
- **53 page-level routes** across `/home`, `/courses`, `/library`, `/vocabulary`,
  `/grammar`, `/progress`, `/speaking`, `/meet`, `/exams`, `/community`, `/live`,
  `/explore`, `/admin`, `/settings`, plus 40+ sub-pages.
- **Design system**: dark + blue accent, Tailwind, `PageHeader` + `AIGate` +
  `LanguageSection` shared primitives. Verified visually in `vite renderer`.
- **State**: Zustand store (`useAppStore`) — auth flags, role, profile, AI cfg.
- **Routing**: HashRouter via React Router; `RequireRole` gates `/teacher/*`
  and `/admin/*`; `usePostBootRedirect` funnels signed-out users to `/signin`.
- **Onboarding**: Welcome → Language picker → ModelCheck → Goals → Interests
  → CEFR placement → Complete.

### Backend (mock — local only)
- `services/backend/local.ts` — full `Backend` contract, persisted to
  `localStorage` under `speakai.backend.v1`. Implements users, courses, units,
  lessons, enrollments, posts (7 kinds), follows, likes, saves, live streams,
  announcements, notifications.
- **⚠️ Single-device only.** Everything a user creates stays on their machine.
  No cross-user feed, no shared courses, no real likes/follows across devices.

### AI (catalog only — not wired to real APIs)
- 8 providers catalogued with strengths, weaknesses, pricing, free-tier info,
  token-acquisition steps: Gemini, Claude, OpenAI, DeepSeek, Grok, Mistral,
  Groq, OpenRouter.
- Settings → AI lets the user paste a token and activate a provider.
- `AIGate` lockscreens `/speaking`, `/ai-tutor`, `/exams/ielts/speaking` when
  no provider is configured.
- **⚠️ No real API call has been made yet** — the IELTS simulator is scripted,
  conversation mode still calls the old Ollama path.

### What's missing for a real, multi-user, AI-powered app
The rest of this doc.

## 1 · Phases (top-down)

| # | Phase | Goal | Hard dep |
|---|---|---|---|
| **P1** | Real AI integration | Each provider routes real chat/vision/STT calls | provider tokens from user |
| **P2** | Real auth (Clerk) | Cross-device identity; real user records | Clerk project |
| **P3** | Supabase backend | Cross-user feed, courses, likes, enrollments persist server-side | Supabase project |

| **P5** | LiveKit realtime | Live streaming + OmeTV WebRTC face-to-face | LiveKit project |

| **P10** | Mobile + polish | iOS/Android via Capacitor, perf, A11Y, i18n | all above |

Each phase below has concrete tasks with file paths and the matching
`Backend` contract methods they touch.

## P1 — Real AI integration

**Outcome**: every AI-dependent feature calls the configured cloud provider
and streams real responses.

### P1.1 — Provider routers
- [ ] `services/ai/providers/gemini.ts` — wrap `@google/generative-ai`,
      streaming text + multimodal audio in.
- [ ] `services/ai/providers/anthropic.ts` — wrap `@anthropic-ai/sdk`,
      streaming with prompt caching enabled by default.
- [ ] `services/ai/providers/openai.ts` — Chat Completions + Realtime API
      shim for the AI tutor voice call.
- [ ] `services/ai/providers/openrouter.ts` — single endpoint covers all
      OpenAI-compatible providers (DeepSeek, Groq, Mistral, Grok via OR).
- [ ] `services/ai/router.ts` — dispatch on `useActiveAI()`. Single async
      generator interface: `chat(messages, opts): AsyncIterable<string>`.

### P1.2 — Wire the existing pages
- [ ] `ConversationMode` (Speaking) — replace Ollama call with router.
- [ ] `AITutorPage` — real streaming response per spoken turn.
- [ ] `IeltsSpeakingSimPage` — examiner questions generated by LLM from
      a system prompt that captures IELTS rubric. Answer evaluation also LLM.
- [ ] Vocabulary translations — call LLM for word-by-word translation
      on demand.
- [ ] Writing rubric — LLM-graded essay scoring with criteria breakdown.

### P1.3 — Speech (STT + TTS)
- [ ] Browser Web Speech for STT in dev (already in code).
- [ ] OpenAI Whisper API path as cloud STT fallback.
- [ ] OpenAI Realtime API or ElevenLabs for natural TTS in the AI tutor.

### P1.4 — Cost guardrails
- [ ] In-app token estimator + monthly spend display per provider.
- [ ] Hard cap setting (e.g. "stop calling AI after $5/month").
- [ ] Auto-fallback to cheaper model when daily quota approaches.

## P6 — Real scoring engines

### P6.1 — Pronunciation (wav2vec2)
- [ ] Python sidecar `services/sidecars/wav2vec2-server.py`. Loads
      `facebook/wav2vec2-large-960h` (or multilingual variant for non-EN
      targets). Listens on a local port.
- [ ] Renderer streams audio chunks → sidecar returns phoneme-level
      confidence + per-word score.
- [ ] `PronunciationPage` shows real scores in the existing UI.

### P6.2 — Listening (Whisper)
- [ ] Same sidecar pattern, or call OpenAI Whisper API directly when
      a provider is configured.
- [ ] IELTS Listening exam grades from the transcribed answer.

### P6.3 — Writing rubric (LLM)
- [ ] System prompt that captures the IELTS Writing band descriptors.
- [ ] Returns `{ overallBand, task: 0-9, coherence: 0-9, lexical: 0-9,
      grammar: 0-9, feedback: string }`.
- [ ] Used by ExamMock for IELTS/TOEFL Writing + by Feedback Exchange.

### P6.4 — Speaking band score (multimodal)
- [ ] In the IELTS Speaking sim, audio is sent to a multimodal model
      (Gemini 2.5, GPT-4o realtime) for joint pronunciation + content
      grading.
- [ ] Returns the 4-criterion band breakdown the result screen already
      renders.
